Comparison  Algorithm

For each  file-to-file comparison,  compute   the    longest-common-subsequence  (LCS)  on  a character-by-character  basis. That  is,   we    want  to  find  the longest  sequence of  characters   that    occurs  in  both  files, in  the same  order,   with    gaps  allowed. ̢̡̢̡̠̦̩̣

You  should  be able  to determine  how   to    read  individual  file  characters, but  the LCS  algorithm   is    a  little  complex.  Here’s a  version of  the   classic    “dynamic  programming”  solution: ̢̡̢̡̠̦̩̣

LCS  Dynamic Programming  Algorithm ̢̡̢̡̠̦̩̣

double calcSimilarity(file1: array of chars, file2: array of chars)

    subsolutions =  2D array of integers, size (file1.length + 1) x (file2.length + 1)
    directionLeft = 2D array of booleans, size (file1.length + 1) x (file2.length + 1)
    fill first row and first column of subsolutions with zeros

    for i := 1 to file1.length (inclusive)    
        for j := 1 to file2.length (inclusive)

            if file1[i - 1] == file2[j - 1]
                subsolutions[i][j] = subsolutions[i - 1][j - 1] + 1

            else if subsolutions[i - 1][j] > subsolutions[i][j - 1]
                subsolutions[i][j] = subsolutions[i - 1][j]
                directionsLeft[i][j] = true

            else
                subsolutions[i][j] = subsolutions[i][j - 1]
                directionLeft[i][j] = false

            end-if            
        end-for
    end-for

    matches = 0
    i = file1.length
    j = file2.length

    while i > 0 and j > 0    
        if file1[i - 1] == file2[j - 1]
            matches += 1
            i -= 1
            j -= 1

        else if directionLeft[i][j]        
            i -= 1

        else        
            j -= 1

        end-if
    end-while

    return (matches * 2) / (file1.length + file2.length)

Note ̢̡̢̡̠̦̩̣

“matches” is  the   length    of  the  longest-common-subsequence.  This is  then used  to   calculate    a  “similarity”  value  between 0  and 1. ̢̡̢̡̠̦̩̣

Note ̢̡̢̡̠̦̩̣

There  are   more    efficient  algorithms  than  this, and  even tweaks  to   this    algorithm  that  could  make it  less wasteful.  You   can    try  to  implement  a better  solution if  you   want,    but  algorithm  design  isn’t really  the aim  of   the    assignment. ̢̡̢̡̠̦̩̣

    • If both  files   are    empty,  the  similarity  score cannot  be calculated,  because   you    would  have  0  divided by  0. However,  that   shouldn’t    happen  because  only  non-empty files  should be  compared. ̢̡̢̡̠̦̩̣

      ̢̡̢̡̠̦̩̣

    • If   both    files  contain  completely  different sets  of characters  (i.e.,   they    have  no  characters  in common  at all),  the   similarity    score  will  be  0. ̢̡̢̡̠̦̩̣

      ̢̡̢̡̠̦̩̣

    • For two  files containing  1000   random    characters,  the  similarity  score will  be low,  but   is    very  unlikely  to  be 0. ̢̡̢̡̠̦̩̣

      ̢̡̢̡̠̦̩̣

    • For  two identical  files,   the    similarity  score  will  be 1.  (This would  also   be    the  result  if  you compared  one file  to   itself,    though  that  isn’t  supposed to  happen.) ̢̡̢̡̠̦̩̣

      ̢̡̢̡̠̦̩̣

    • If one  file   contains    exactly  half  of  the other  file (including  if   the    second  file  is  just two  copies of  the   first    file  concatenated  together),  then the  similarity score  will   be    0.66667. ̢̡̢̡̠̦̩̣

      ̢̡̢̡̠̦̩̣

    • If  one  file  contains exactly  one third  of   the    other,  then  the  similarity score  will be  0.5. ̢̡̢̡̠̦̩̣

      ̢̡̢̡̠̦̩̣
